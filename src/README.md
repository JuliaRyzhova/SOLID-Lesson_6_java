На мой взгляд суть программы состоит в сохранении пользователя и формировании отчета для конкретного пользователя.

Класс User нарушает 1-ый принцип SOLID - единственной ответственности и 5-ый принцип в отношении инверсии зависимотей.
Нарушение состоит в том, что он не только представляет собой объект класса, но и берет на себя
задачу по сохранению пользователя и формированию отчета.
Класс Persister также нарушает эти принципы.

Для решения этих проблем:
1. Создала новый класс Report в котором прописала метод report() принимающий в качестве параметра некий экзмепляр типа User.
2. Создала интерфейс Savable с методом save(). 
3. Создала класс Saver и имплементировала ему интерфейс Savable. 
4. В классе User оставила только ту информацию, которая имеет отношение к данному классу.

В реализации данной задачи, старалась придерживаться паттерну MVC.
